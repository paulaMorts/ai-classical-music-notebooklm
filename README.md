# Classical Second Brain with NotebookLM

## Objective

This project explores how generative AI and NotebookLM can function as a knowledge augmentation system for studying classical music history, composers, pianists, and piano interpretation.

The repository documents an AI-assisted learning workflow designed to organize, synthesize, and retrieve historical musical knowledge through contextual prompting and semantic organization.

The project combines:

- Prompt Engineering
- Knowledge Organization
- Semantic Retrieval
- Contextual Synthesis
- AI-assisted Learning
- Historical Music Analysis
- Multimodal Learning Assets

---

## Workflow Architecture

Sources → NotebookLM → Prompt Engineering → Contextual Synthesis → Knowledge Organization → Learning Assets

## Technical Insights

This project demonstrated how contextual prompting and semantic retrieval workflows can improve long-term learning and historical knowledge synthesis using generative AI systems.

Key observations:

- Comparative prompts generated richer contextual responses
- Narrative prompts improved historical continuity
- Structured prompts reduced shallow summaries
- Context-aware prompting improved semantic linking between composers and musical movements
- AI-generated synthesis accelerated knowledge organization workflows

---

## Challenges and Troubleshooting

Throughout the project, several prompt engineering and contextual retrieval challenges were identified:

- Generic prompts produced shallow or repetitive responses
- Historical prompts required contextual specificity
- Comparative prompting improved thematic coherence
- Long-form prompts generated stronger semantic relationships
- Broader prompts produced more interconnected historical narratives
- Prompt refinement significantly improved response quality
